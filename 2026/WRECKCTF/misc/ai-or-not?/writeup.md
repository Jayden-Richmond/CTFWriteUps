# ai-or-not? Writeup

**CTF EVENT:** WRECKCTF 2026
<br>**Category:** Misc
<br>**Points:** 174pts

## 📝 Challenge Description
> Our research team just deployed ARIA — our most advanced AI assistant yet. Third generation, fully trained, supposedly impenetrable.
> <br> I also can't quite remember where I wrote down the override token...

## Background Information
When starting the challenge, I was presented with a link to a website called ARIA. ARIA was presented as an intelligent chatbot with adaptive reasoning. Going into this challenge, my assumption was to find an override token that could modify ARIA's behavior to give me a flag. As is the case with any web challenge, the first step should always be to inspect the website's behavior.

## Reconnaissance
After looking at the behavior of ARIA and feeding it a multitude of inputs, I realized I was getting premade static phrases instead of more creative responses from an LLM. This meant I could not do anything to convince ARIA to give me the flag or override token. After understanding this concept, I moved on to inspect the page source index for the website. 

While inspecting the page source of the index, I found a deployment checklist hidden in the HTML comments. This leaked some highly sensitive internal deployment information:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>ARIA — AI Assistant</title>
```

Using this information, you can draw several conclusions. This is a checklist of tasks that needed to be done to get ARIA working as expected. Two important tasks have yet to be implemented as denoted by "PENDING." These tasks tell us that the legacy admin interface has yet to be disabled and that diagnostic tooling can be found at `/download/aria-diagnostic`. As with any challenge or site with notes like this, we are going to follow the breadcrumbs until we reach something of interest or a dead end. I navigated to the `/download/aria-diagnostic` endpoint and received a `.pyc` file.

If you are unfamiliar with `.pyc` files, they are files that contain compiled Python bytecode. They are generated by a Python interpreter when a Python script is run. Furthermore, we can get the Python source code from a `.pyc` file, and vice versa. When I did this challenge, I was in a rush to complete challenges as quickly as possible, so I used https://www.pylingual.io/ to convert my `.pyc` file into readable Python code that could be analyzed for something of interest.

Now, I am not going to display all of the Python code, but rather some important parts I found within it. Remember the hint from earlier that said the override token was written somewhere? This is where it was written.

``` python
OVERRIDE_TOKEN = _reconstruct_override()

_XOR_MASK = 170
_CIPHER_TEXT = bytes([235, 248, 227, 235, 144, 144, 238, 227, 235, 237, 144, 144, 229, 252, 239, 248, 248, 227, 238, 239, 144, 144, 203, 157, 204, 153, 152, 155, 147, 207])
def _reconstruct_override() -> str:
    return ''.join((chr(b ^ _XOR_MASK) for b in _CIPHER_TEXT))
```
Looking at this, we face the final hurdle of acquiring the `OVERRIDE_TOKEN`. Whoever wrote ARIA has made the token slightly harder to see. They are preventing it from being easily discovered via simple searches by implementing string obfuscation. The token is encoded using a single-byte XOR cipher. The script stores the ciphertext as a byte array and uses a hardcoded XOR mask of `170` to decrypt it at runtime. Because XOR is a symmetric operation, this can be easily reversed to reveal the plaintext token.

## Background on What an XOR Mask is
Before I continue my process, I wanted to go over what an XOR operation and an XOR mask are for people new to computer science. (For people familiar with the term, feel free to skip ahead). In computer science, XOR (Exclusive-OR) is a fundamental logic operation. When given two lines of bits and we wish to compare them with an XOR operation, we line them up and compute. If the bits match, then the resulting bit from the operation is a 0. If they don't match, for example, a 0 and a 1, then the result is a 1. This is why it's exclusive—you can only have 1 or the other, but never both. I implore you to look into bitwise operations to see how they work, because they are an important foundation.

To help illustrate the concept, I am going to give an example of what an XOR operation looks like. A mask is any line of bits we are using to modify a set of bits we have. In the case of the example, assume the 2nd line of bits is the mask.

```text
  01000001  (Original Data: the letter 'A' / 65)
^ 10101010  (XOR Mask: the number 170)
------------------------------------------------
  11101011  (Ciphertext: Scrambled data / 235)
```

## The Script
With that information out of the way, I began deciphering the cipher with a python for loop that would perform the XOR operation on every byte in it with `170` as a mask. I also wanted these bytes converted into readable plaintext, so each byte was converted into a character.
``` python
for i in _CIPHER_TEXT:
    print(chr(i ^ _XOR_MASK), end="")
```
Running this you find out that the Override phrase for ARIA is 

`ARIA::DIAG::OVERRIDE::a7f3219e`

So now we have our token, and the hard part is out of the way. I inserted the token back into ARIA, and the result was this:

## Flag and Conclusions

Finishing the challenge resulted in collecting the flag:

`wreck{b34c0n_l3g4cy_n3v3r_d3pr3c4t3d}`

The most important lesson to take from this challenge when looking at future CTFs are to always analyze comments and documentation. In the real world, people tend to reveal some information about their systems and methods through comments and documentation. In this case, the documentation revealed sensitive data that we used to gain an override token. This challenge, while being relatively simple in difficulty, can be used as a baseline for investigating other web or misc challenges. Always play with the behavior of the site, then take a moment to analyze the information you are given. After this, once you have developed some leads, you can begin exploring until you find something useful. Rinse and repeat!

Overall, this was a fun introductory challenge.
